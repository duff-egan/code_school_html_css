Font Face

Using @font-face, we have the ability to provide online fonts for use on our websites.

@font-face {
  font-family: 'DroidSerifRegular'
  src: url('DroidSerif.eot');
  font-style: normal;
  font-weight: normal;
}
body {
  font-family: DroidSerifRegular, Georgia, "Times New Roman", serif;
}

To add a bold version of the font 'DroidSerifRegular', DroidSerifBold', using the local version at 'DroidSerifBold.eot', we'll create a new @font-face declaration, but set it up so we can use 'font-weight' to change the weight.

@font-face {
  font-family: 'DroidSerifRegular'
  src: url('DroidSerif.eot');
  font-style: normal;
  font-weight: bold;
}

Because h1 will already use font-family from the body element, so we'll only need to set the font-weight here.

h1 {
  font-weight: bold;
}

Code School gives a .eot font specification in their example, but according to StackOverflow, it is an old and poor format to use, and the .woff file type is the best.

According to W3Schools.com, "Internet Explorer 8 and earlier, do not support the @font-face rule with the WOFF format (only support for EOT format)."

Additionally, the @font-face rule requires that the fonts be present on the client computer and instructs the browser to download the font from where it's hosted. This seems a little outdated and inefficient to me. However, it also appears to have the deepest browser support at this time.

It seems like it would be much easier to use Google Fonts. In the <head> of the page, we put something like:

<link rel="stylesheet" type="text/css"
          href="https://fonts.googleapis.com/css?family=Tangerine">

Then, in my CSS stylesheet:

.font-face-example-chp5-google-fonts {
        font-family: 'Tangerine', serif;
        font-size: 48px;
      }

CSS3 Transforms

Using the transform property in CSS3, we can translate, rotate, scale, and skew various elements in CSS.

You can create a 2D translation using transform:

.element {
    transform: translate(40px, 10px);
}

Which creates:

Original
2D Translate

We're setting a 40px x-axis value, and a 10px y-axis value.

The translate function takes two arguments:

  • A translation value for the x-axis which can be either a length or a percentage.
  • A translation value for the y-axis which can be either a length or a percentage. If we don't specify this value, it sets to 0 by default.

The x and y values can also be used individually:

.element {
    transform: translateX(20px);
}
.element {
    transform: translateY(30px);
}

With rotate, you can rotate an element clockwise around its origin by the specified angle:

.element {
    transform: rotate(45deg);
}

Which creates:

Original
Rotate

With scale, you can do a 2D scale by a specified unitless number:

.element {
    transform: scale(1.2);
}

Which creates:

Original
Scale

The scale transform takes two arguments:

  • The value for the x-axis, which again is a unitless number.
  • The value for the y-axis, which is a unitless number. If we don't specify this value, it will default to what is specified in the first argument.

The x and y values can also be used individually:

.element {
    transform: scaleX(1.2);
}
.element {
    transform: scaleY(0.3);
}

With skew, an element is skewed around the x or y axis by the angle specified. Unlike the other transforms, skew only allows for skewing each axis individually:

.element {
    transform: skewX(25deg);
}

CSS3 Transitions

CSS3 provides transitions, which allow us to transition between two states of a specified element.

Example usage of the shorthand transition property:

.element {
    transition: <property> <duration> <timing-function> <delay>;
}
  • property: e.g., background color, font color, etc.
  • duration: the duration value in seconds. The default is "0s" (0 seconds).
  • timing-function: the timing of the function itself, which has a few different keywords:
    • ease
    • ease-in
    • ease-in-out
    • linear
    • cubic-bezier
    • step-start
    • step-end
    • steps()
  • delay: The amount of time to wait between the change that is being requested on a specific property, and the start of the transition.

We can set the transition values individually as well:

.element {
    transition-property: background-color;
    transition-duration: 1.0s;
    transition-timing-function: ease-in-out;
    transition-delay: 0.1s;
}

We can transition multiple properties at once (in this case the background color and font color) by using all as the transition-property:

.element {
    background-color: #5f9ea0;
    transition: all 1.0s, ease-in-out;
}
.element:hover {
    background-color: #42FF21;
    color: #ffffff;
}

Progressive Enhancement

The term progressive enhancement refers to the use of newer features that add to the experience in modern browsers that support those features, but doesn't detract from the experience in older browers.

For example, if we had an element with the border-radius and box-shadow properties...

.element {
    border-radius: 10px;
    box-shadow: 3px 3px 1px rgba(0, 0, 0, 0.75);
}

In older browers that don't support the border-radius and box-shadow properties, it wouldn't have those features, but it would still be usable, which is what progressive enhancement is all about:

Progressive enhancement is important to keep in mind because we want to make sure that we're creating experiences that degrade gracefully for browsers that don't support a lot of the features that have been covered here.
