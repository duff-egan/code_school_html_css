New Input Types

HTML5 provides several new input types:

  • Search
  • Email
  • URL
  • Tel
  • Number
  • Range
  • Date
  • Month
  • Week
  • Time
  • Datetime
  • Datetime-local
  • Color

NOTE: If a browser doesn't support one of these input types, it will default to "text."

Search

The input element with a type attribute whose value is "search" represents a one-line plain-text edit control for entering one or more search terms - W3C specification.

New Form Elements

HTML5 provides new form elements:

  • Datalist
  • Keygen
  • Output

Here we're only going to focus on the datalist element because it's the most commonly used one.

Datalist

The "datalist" element represents a set of option elements that represent predefined options for other controls - W3C specification.

The "id" and "list" values need to be the same for them to link together properly. The "option values" will be the values available to our input:

<input type="text" list="browsers" />
<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Internet Explorer">
    <option value="Opera">
    <option value="Safari">
</datalist>

Creates:

The input looks like a regular text input, but when we start typing, you can see that we get auto-complete suggestions based on the option values that we specified within the datalist element. Also, when the cursor is in the input box, a dropdown arrow appears at the right allowing us to select from the available options with our mouse.

New Form Attributes

HTML5 provides several new form attributes:

  • Placeholder
  • Autofocus
  • Autocomplete
  • Required
  • Pattern
  • List
  • Multiple
  • Novalidate
  • Formnovalidate
  • Form
  • Formaction
  • Formenctype
  • Formmethod
  • Formtarget

Here we're going to focus primarily on the placeholder, autofocus, required, and pattern attributes.

Placeholder

The "placeholder" attribute allows you to specify a message that is shown inside the input. That message is then hidden when the user starts typing, but then returns when focus is lost on the input (when the input is blank).

We could use the "value" attribute in an input field, but when we do that, it requires the user to clear out that value before entering information into the field:

<input type="text" value="Enter your email..." />

Creates:

By using the "placeholder" attribute, we make it easier for the user. The text in the input box doesn't have to be cleared, and the user is able to immediately type information into the field:

<input type="text" placeholder="Enter your email..." />

Creates:

Autofocus

The "autofocus" attribute will automatically focus the specified input when the page is rendered. This is something that used to have to be done in JavaScript. But now we just have to add one simple attribute to get the effect.

<input type="text" autofocus />

Required

If you add the new HTML5 "required" attribute to an input, when the form is submitted, the user will be notified of an error if the field is left blank.

Like the "autofocus" attribute, all we have to do is add the "required" attribute to our input.

<input type="text" required />

Pattern

The "pattern" attribute accepts a JavaScript regular expression that can be used to validate a form field to match the pattern.

In this example we're matching three digits:

<input type="text" pattern="[0-9]{3}" />

When the form is submitted, if the user's entry doesn't match the pattern exactly, they're notified of an error until the pattern is matched.
