HTML5 Elements

Section

The section element represents a generic document or application section - W3C specification.

This is very similar to a div element. However a div has no semantic meaning, whereas the section element does. It's used for grouping together thematically related content.

In his book, HTML5 For Web Designers, Jeremy Keith tells us "You might want to replace some of your div elements with section elements, but remember to always ask yourself, 'Is all of the content related?'"

While all section elements don't necessarily have to contain a heading, most generally do.

Prior to HTML5 we probably would have used a div with a class or id of "section:"

<div class="section">
    <h2>The Gallery</h2>
    <!-- ... -->
</div>

In HTML5 though we can instead use the section element to wrap our markup which makes our document much more semantic:

<section>
    <h2>The Gallery</h2>
    <!-- ... -->
</section>

Header

A group of introductory or navigational aids - W3C specification.

It's important to note that you can include more than one header on a page - it's not just limited to the top portion of your site.

The header usually appears at the top of a document or section, but is defined by its content rather than its position.

Prior to HTML5 we probably would have used a div with a class or id of "header:"

<div class="header">
    <!-- ... -->
</div>

In HTML5 though we can instead use the header element to wrap our markup which makes our document much more semantic:

<header>
    <!-- ... -->
</header>

Remember, you can have header elements in different parts of the page, and inside of other HTML5 elements:

<section>
    <header>
        <h1>The heading of the section</h1>
        <p>This is content in the header</p>
    <header>
    <p>This is some information within the section.</p>
</section>

Footer

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element - W3C specification.

Like the header, the footer element is not position-dependent. It should describe the content it is contained within.

Prior to HTML5 we probably would have used a div with a class or id of "footer:"

<div class="footer">
    <!-- ... -->
</div>

In HTML5 though we can instead use the footer element to wrap our markup which makes our document much more semantic:

<footer>
    <!-- ... -->
</footer>

Like the header, we can also put the footer element inside other HTML5 elements:

<section>
    <header>
        <h1>The heading of the section</h1>
        <p>This is content in the header</p>
    </header>
    <p>This is some information within the section.</p>
    <footer>
        <p>By "Author Name"</p>
    </footer>
</section>

Aside

Initially, the HTML5 spec defined the aside element as being "tangentially related to the content surrounding it," which essentially meant that it was not directly related to the content, but expands upon it.

However, the aside element now covers various contexts:

  • When used within an article element, the aside contents should be related to that particular article it is contained within.
  • When used outside an article element, the aside contents should be specifically related to the site.

An aside element is most appropriate when it is used to represent content that is not the primary focus of an article or page, but it is still related to the article or page.

Prior to HTML5 we probably would have used a div with a class or id of "sidebar:"

<div class="sidebar">
    <!-- ... -->
</div>

In HTML5 though we can instead use the aside element to wrap our markup which makes our document much more semantic:

<aside>
    <!-- ... -->
</aside>

Like the header and footer elements, we can also next the aside element inside other HTML5 elements such as the section:

<section>
    <header>
        <h1>The heading of the section</h1>
        <p>This is content in the header</p>
    </header>
    <p>This is some information within the section.</p>
    <aside>
        <p>Some secondary information.</p>
    </aside>
    <footer>
        <p>By "Author Name"</p>
    </footer>
</section>

Nav

The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links - W3C specification.

The nav element is intended for "major navigation."

Prior to HTML5 we probably would have used an unordered list with a class or id of "nav:"

<ul class="nav">
    <!-- ... -->
</ul>

In HTML5 though we can instead use the nav element to wrap our markup which makes our document much more semantic:

<nav>
    <ul>
        <!-- ... -->
    </ul>
</nav>

Article

The article element represents a complete, or self-contained, composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication - W3C specification.

The article element is another type of section. It is used for self-contained related content.

Determining if a particular piece of content is "self-contained:" Ask yourself if you would sydicate the content in an RSS or Atom feed.

Some uses for the article tag:

  • A blog post
  • A news story
  • A comment on a post
  • A review

Prior to HTML5 we probably would have used a div with a class or id of "article:"

<div class="article">
    <!-- ... -->
</div>

In HTML5 though we can instead use the article element to wrap our markup which makes our document much more semantic:

<article>
    <!-- ... -->
</article>

Main

The main element represents the main content of the body of a document or application - W3C specification.

The main content area consists of content that is directly related to or expands upon the central topic of a document or central functionality of an application - W3C specification.

  • Do not include more than one main element in a document
  • Do not include the main element inside of an article, aside, footer, header, or nav element

Prior to HTML5 we probably would have used a div with a class or id of "main:"

<div class="main">
    <!-- ... -->
</div>

In HTML5 though we can instead use the main element to wrap our markup which makes our document much more semantic:

<main>
    <!-- ... -->
</main>

Figure/Figcaption

The figure element represents a unit of content, optionally with a caption, that is self-contained, that is typically referenced as a single unit from the main flow of the document, and that can be moved away from the main flow of the document without affecting the document's meaning - W3C specification.

A common use of the figure tag is for an image within an article:

<figure>
    <img src="image.jpg" alt="My Picture" />
</figure>

The figcaption element represents a caption or legend for a figure - W3C specification.

Example usage of the figcaption tag:

<figure>
    <img src="image.jpg" alt="My Picture" />
    <figcaption>This is a caption for the picture.</figcaption>
</figure>

Time

The time element represents either a time on a 24 hour clock, or a precise date in the proleptic Gregorian calendar, optionally with a time and a time-zone offset - W3C specification.

Example usage of the time tag:

<time>2013-09-16</time>

However, we want the format "mm/dd/yyy." So we put that format in between our time tags:

<time>09/16/2013</time>

But in order to use that date format, we need to make an important change. We have to add the datetime attribute to our time element with the desired date format:

<time datetime="2013-09-16">09/16/2013</time>

This allows us to use any relevant content to the date or time within the time element. If we don't use that datetime attribute on the time element, the content must either be a valid date, time, or a precise datetime.
