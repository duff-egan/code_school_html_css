The 'for' and 'id' Attributes

There are also some attributes you should add to your labels and inputs in order to connect them together at the HTML level.

The value for the for attribute in the label should be the same as the value of the id attribute in an input field to associate the label and input.

Each for/id pair has to be unique on the page.

<form>
    <label for="recipe-name">Recipe Name</label>
    <input type="text" id="recipe-name">
    <input type="submit" value="Click to Submit">
</form>

Using a textarea Instead of an Input

input tags should be used for short or single-line user input.

The textarea tag is used for multi-line user input. Unlike the input tags, the textarea tag uses opening and closing tags. Also, a textarea tag doesn't require a "type" because the only thing that can go in there is text.

<form>
    <label for="ingredients">Ingredients</label>
    <textarea id="ingredients"></textarea>
    <input type="submit" value="Click to Submit">
</form>

Styling Forms

labels and inputs are inline-level tags, but it usually makes sense to display one on top of the other like block-level instead of side-by-side.

.foundations-example-chp5-block label,
        .foundations-example-chp5-block input {
    display: block;
}

<div class="foundations-example-chp5-block">
    <form>
        <label>Recipe Name</label>
        <input type="text">
        <input type="submit" value="Click to Submit">
    </form>
</div>

Once you've got both tags displaying as block-level, you can style them just like you would any other block-level tag. This code will add margin to the bottom of the label element now:

.foundations-example-chp5-block label,
        .foundations-example-chp5-block input {
    display: block;
}
.foundations-example-chp5-margin label {
    margin-bottom: 5px;
}

<div class="foundations-example-chp5-block

    <form>

        <input type="text">
        <input type="submit" value="Click to Submit">

When we have multiple inputs, we can adjust the margins between them as well as the width of the input boxes in order to make them consistent.

.foundations-example-chp5-block label,

.foundations-example-chp5-margin label {

Attribute Selectors

However, when we set the width on all of the input tags, we made the "Click to Submit" button the same width as the input boxes, because it is also an input element. But we don't really want the submit button to be so wide.

To remedy this, we can use an attribute selector. Attribute selectors are a way to style a tag based on one of its attributes

We write the name of the tag (in this case it's "input"), then inside of square brackets we write the name of the attribute (in this case it's "type"), and the attribute's value (in this case it's "submit").

My HTML code doesn't change. I just create another CSS class and add the attribute selector to it: 

.foundations-example-chp5-width input[type=submit] {
    width: 120px;
}

Styling Inputs

We can do something similar to other input types.

The container around an input is actually just a border, so you can style it with the border property.

.foundations-example-chp5-input-styling input[type=text] {
    border: 3px solid #7facaa;
}

<div class="foundations-example-chp5-block
    foundations-example-chp5-margin
        foundations-example-chp5-width
            foundations-example-chp5-input-styling-1">
    <form>
        <label>Recipe Name</label>
        <input type="text">
        <label>Your Name</label>
        <input type="text">
        <label>Your Email</label>
        <input type="email">
        <input type="submit" value="Click to Submit">
    </form>
</div>

We've adjusted the width of our input fields by using the width property, but we can't just use the height property to adjust the height of text input fields.

Instead, we have to use a combination of changing the font size, or padding. In other words, to adjust the height of the input, we style the text inside.

For example, if we change the font-size, the text box will automatically adjust its size to contain the text. Also, if we want to make the input larger beyond just increasing the text size, we can add padding to the text box.

.foundations-example-chp5-input-styling-2 input[type=text] {
    border: 3px solid #7facaa;
    font-size: 24px;
    padding: 10px;
}

<div class="foundations-example-chp5-block
    foundations-example-chp5-margin
        foundations-example-chp5-width
            foundations-example-chp5-input-styling-2">
    <form>
        <label>Recipe Name</label>
        <input type="text">
        <label>Your Name</label>
        <input type="text">
        <label>Your Email</label>
        <input type="email">
        <input type="submit" value="Click to Submit">
    </form>
</div>

Creating a Separate Style for a checkbox Input

It looks awkward if the newsletter label and input are on separate lines since the checkbox is so small, so we can use attribute selectors again to make just this input and label display inline.

The problem is that we've already set all of our labels and inputs to block which is forcing the checkbox to move down to a new line.

So, again with attribute selectors, we can select all inputs with the type of checkbox, and all labels with a for attribute of newsletter and we can set the display back to inline. 

.foundations-example-chp5-checkbox-inline input[type=checkbox],
    .foundations-example-chp5-checkbox-inline label[for=newsletter] {
    display: inline;
}

<div class="foundations-example-chp5-base
    foundations-example-chp5-block foundations-example-chp5-margin
        foundations-example-chp5-checkbox-inline">
    <form>
        <label for="newsletter">Get Newsletter?</label>
        <input type="checkbox" id="newsletter">
    </form>
</div>
